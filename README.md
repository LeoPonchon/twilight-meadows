# **Projet Unity 2D Pixel Art**

Bienvenue dans le projet interne de développement d'un jeu 2D en pixel art. Ce projet est en cours de conception et a pour objectif final une commercialisation. Bien que le thème du jeu ne soit pas encore défini, l'accent est mis sur la qualité des graphismes et l'innovation dans le gameplay 2D.

## **Objectifs du Projet**

Le but de ce projet est de créer un jeu 2D engageant et visuellement distinct grâce à l'utilisation de graphismes en pixel art. Le thème et les mécaniques de gameplay seront décidés en collaboration avec l'équipe, mais l'idée est d'explorer divers concepts potentiels en termes d'univers, de style artistique et d'interactions.

### **Progrès Actuel**

- **Définir le thème et l'univers du jeu** : Jeu d'aventure dans un monde coloré, avec des enigmes, du combat, des quêtes et de l'interaction avec des PNJ.
- **Mécaniques de Base** : Système de mouvement 2D et gestion des collisions.
  
### **Prochaines Étapes**

- **Étendre le gameplay** : Intégrer de nouvelles mécaniques de jeu selon le thème choisi (combat, exploration, puzzles, etc.).
- **Graphismes Pixel Art** : Mise en place des bases visuelles, avec un style pixel art rétro et soigné.
- **Animations** : Premières animations du personnage principal, y compris les cycles de marche et d'attaque.
- **Améliorations techniques** : Optimisation des animations, gestion des états du joueur, ajout de nouvelles fonctionnalités.
- **Conception audio** : Création et intégration de la bande sonore et des effets sonores (collaboration avec des compositeurs/musiciens).

### **Prérequis**

- Unity (version : 2022.3.13f1)
- Un éditeur de code (Visual Studio ou autre compatible avec Unity)

### **Installation**

1. Clonez le dépôt depuis notre serveur interne :
   
   ```bash
   git clone https://github.com/LeoPonchon/projet-unity.git
   ```

2. Ouvrez Unity et chargez le projet en sélectionnant le dossier cloné.

3. Assurez-vous d'importer tous les packages nécessaires dans Unity si des messages d'alerte apparaissent.

## **Structure du Projet**

- **Assets/** : Contient les ressources graphiques, les animations, les sons et autres assets utilisés dans le projet.
- **Scripts/** : Tous les scripts C# liés aux mécaniques du jeu (contrôles du joueur, gestion des ennemis, etc.).
- **Scenes/** : Contient les scènes du jeu (ex. : menus, niveaux en cours de création).
- **Resources/** : Répertoire pour les ressources utilisées dans les scripts, comme les fichiers de configuration ou les prefabs partagés.

## **Gestion du Projet**

Le développement est organisé en sprints avec des itérations régulières pour tester les fonctionnalités. Les contributions se font via des branches Git dédiées aux différentes fonctionnalités. Les décisions sur le gameplay et le design se prennent en équipe, avec des réunions hebdomadaires pour ajuster la direction du projet.

### **Comment Contribuer**

1. Avant de commencer à travailler, assurez-vous de consulter le tableau de bord du projet sur notre gestionnaire de tâches (ex. : Trello, Jira, etc.).
2. Créez une nouvelle branche pour chaque fonctionnalité ou correction de bug :
   
   ```bash
   git checkout -b feature/nouvelle-fonctionnalite
   ```

3. Une fois les modifications terminées, soumettez une Pull Request pour révision et test interne.
